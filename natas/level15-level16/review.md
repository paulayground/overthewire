# 배운 점

`$()`: 를 사용해 명령어를 사용한 결과를 가져올 수 있다.

# 풀이 과정 기록

정규표현식이 걸려있는 `grep` 사이에 명령어 injection하여 해결하는 문제라는 것을 파악했다.

제시된 정규표현식을 우회해서 쓸수 있는게 뭐가 있는지 고민하다 답을 못 찾아서 찾아 본 결과 `$()` 사용하여 해결하는 힌트를 얻었다.

하지만 포커스를 `""`을 깨뜨리는 방향으로 잡다보니 `""`안에서 해결할 수 있는 방법을 생각을 못했다.

더 찾아보다가 boolean base blind injection 힌트를 얻고 `$(cut -c 1 /etc/natas_webpass/natas17)`와 같이 비밀번호의 한 자리씩 꺼내서 일치하는 결과의 알파벳을 유추하는 방식을 잡았다.

각 알파벳의 결과와 유추한 결과들을 비교해서 32자리의 비밀번호를 얻었다.
`eqjhjbo[0-9]lfnb[0-9]vwhhb[0-9]s[0-9][0-9]hokh[0-9]tf[0-9]oc`

각 알파벳의 대소문자와 숫자를 대입하는 방식으로 로그인 요청을 해서 풀 수 있긴했지만, 이 방향이 아닌거 같았다.

확인해보니 `grep -i "$(grep ^a /etc/natas_webpass/natas17)" dictionary.txt`와 같이 내부 명령어실행 결과가 비밀번호와 일치한다면 전체 비밀번호를 가져오며, 없을경우 내부적으로는 빈 값을 리턴해 모든 값을 가져오는 방식을 통해 해결할 수 있다는 것을 확인하고 방향을 바꿨다.
